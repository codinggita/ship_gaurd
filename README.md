# ShipGuard OS | Tactical Logistics & Claim Protection Platform

![ShipGuard Banner](https://api.dicebear.com/7.x/shapes/svg?seed=ShipGuard&backgroundColor=0A0A0A)

**ShipGuard OS** is a high-fidelity, tactical logistics management platform designed to bridge the gap between courier services and shippers. It leverages advanced data visualization, real-time telemetry, and AI-driven risk assessment to ensure package safety and streamline the compensation process.

---

## 🚩 Problem Statement

E-commerce sellers and personal shippers face a significant challenge in the current logistics landscape:
- **High Loss Rates**: 3-5% of packages are lost or damaged during courier transit.
- **Bureaucratic Hurdles**: Filing claims requires extensive and often redundant documentation.
- **Extreme Latency**: Courier companies typically take **30-45 days** for claim processing.
- **Compensation Friction**: Legitimate claims are frequently disputed or denied due to a lack of transparent data.
- **Costly Protection**: Traditional insurance options are prohibitively expensive for small-to-medium sellers.
- **Accountability Gap**: No existing mechanism ensures careful package handling or provides real-time accountability for courier partners.

---

## 🛡️ The Solution: Our Approach

ShipGuard OS provides a comprehensive digital terminal that transforms logistics from a "black box" into a transparent, data-driven operation. Our frontend approach focuses on **Actionable Intelligence** and **Accountability**:

### 1. Command_CTR (Central Dashboard)
Provides an immediate bird's-eye view of global traffic flux, recovery ratios, and system integrity. It allows operators to identify anomalies across the entire network at a glance.

### 2. AI Risk Analysis (FleetOps)
Uses predictive modeling to assign risk scores to specific routes and courier nodes. By visualizing variance charts and threat cards, shippers can proactively choose safer transit paths.

### 3. Shipment_Live_Track (Manifests)
A telemetry-heavy interface that tracks individual units with extreme precision. It records transit sequences and cargo specs, creating an immutable "digital breadcrumb trail" for accountability.

### 4. Claims Terminal (Rapid Recovery)
A streamlined, 4-step automated wizard that simplifies evidence submission. By standardizing damage reporting and diagnostic summaries, it significantly reduces the 30-day processing window.

### 5. Alert Center (System Logs)
A terminal-style feed that categorizes events by priority (CRITICAL, WARNING, INFO). It ensures that any breach in handling protocols or shipment delay is flagged in real-time.

## 🚀 Technology Stack

### Frontend (The Terminal)
- **React.js (Vite)**: For a lightning-fast, modular UI architecture.
- **Vanilla CSS & Tailwind**: Custom-engineered "Tactical Dark" design system with glassmorphism and neon accents.
- **Framer Motion**: Powering the immersive "boot-up" sequences and fluid state transitions.
- **Recharts**: High-performance data visualization for traffic flux and risk variance.
- **Lucide React**: Specialized iconography for tactical interfaces.
- **React Helmet Async**: Full SEO optimization for production readiness.

### Backend (The Core)
- **Node.js & Express**: Scalable API architecture for handling high-frequency logistics data.
- **JWT (JSON Web Tokens)**: Secure, stateless authentication for operator access.
- **Middleware Architecture**: Robust route protection and request logging.

### Backend Setup
